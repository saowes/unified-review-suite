# /review — 发起统一审查

使用 `review-orchestrator`(统一审查编排器)技能，对当前改动或我指定的目标执行一次统一审查。

请按以下步骤进行：

1. 先识别审查目标:文档一致性 / 代码质量与安全 / 全量。若我未说明，默认对本次会话的改动做全量审查。
2. 文档线:套用 `document-review-sync-check` 的 L1/L2/L3 方法。
3. 代码线:派起内置子代理 `bugbot`、`security-review`、`code-reviewer`;若环境不支持则降级并标注。
4. 验证收尾:对"已修复/已通过"的结论必须复检后再下结论。
5. 按统一报告模板输出，问题用 P0–P3 分级，并区分"内容问题"与"rollout 问题"。

可选参数(在命令后补充):
- 指定目标路径或目录
- 只跑文档线 / 只跑代码线
- 已知的关联文档、JSON/配置、资源目录
